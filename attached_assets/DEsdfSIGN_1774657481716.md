# Design System Specification: Editorial Aviation Authority

## 1. Overview & Creative North Star
**The Creative North Star: "The Technical Curator"**
This design system is built to convey the unwavering precision of aviation logistics through the lens of a high-end editorial publication. It rejects the "SaaS-standard" look of cluttered grids and heavy borders, opting instead for a spacious, technical elegance. We achieve this by balancing a deep, authoritative navy foundation with "Technical Accents"—amber, emerald, and blue—that mimic the illuminated gauges of a modern flight deck.

The experience is defined by **intentional asymmetry** and **tonal depth**. Rather than containing information within rigid, outlined boxes, we treat the UI as a series of layered, "floating" data surfaces. The layout uses wide margins and sophisticated typography to guide the eye, ensuring that complex registry data feels curated and manageable rather than overwhelming.

---

## 2. Colors
Our palette is rooted in the "No-Line" Rule: **Strictly prohibit 1px solid borders for sectioning.** Physical boundaries are defined solely through background shifts and elevation.

### Core Palette (Material Design Tokens)
- **Primary / Background:** `primary` (#000b25) serves as our navigational anchor, while `background` (#f6fafe) provides a breathable canvas for the main workspace.
- **Surfaces:** Use `surface_container_low` (#f0f4f8) for the base workspace and `surface_container_lowest` (#ffffff) for primary content cards. This creates a natural, soft lift.
- **Technical Accents:**
    - **Emerald:** `on_tertiary_container` (#009d6c) for "Active" or "Safe" states.
    - **Amber:** `on_error_container` (#93000a) / Custom Amber for "Pending" or "Warning."
    - **Blue:** `primary_fixed_dim` (#b3c6f5) for interactive high-contrast moments.

### The "Glass & Gradient" Rule
To elevate the UI beyond standard flat design, floating elements (like active navigation indicators or dropdowns) should utilize **Glassmorphism**. Apply a semi-transparent `surface_variant` with a 12px backdrop-blur. 
*Signature Polish:* For primary CTAs, use a subtle linear gradient transitioning from `primary_container` (#0b2147) to `primary` (#000b25) at a 135-degree angle.

---

## 3. Typography
We utilize **Manrope** across all scales. Its geometric construction feels modern and technical, yet its open counters maintain readability in dense data environments.

- **Display (lg/md):** Reserved for high-impact dashboard summaries. Use `display-md` (2.75rem) with -0.02em tracking to create an authoritative, "cockpit" feel.
- **Headlines (lg/sm):** Used for page titles and major section headers. `headline-sm` (1.5rem) should be semi-bold to establish immediate hierarchy.
- **Title (md/sm):** Specifically for card headers. These act as the primary anchor for information groupings.
- **Body (md/sm):** The workhorse for registry data. Use `body-md` (0.875rem) for most table content to maximize information density without sacrificing legibility.
- **Labels (md/sm):** Used for metadata, breadcrumbs, and status tags. These should be all-caps with +0.05em tracking for a "technical spec" aesthetic.

---

## 4. Elevation & Depth
Depth is achieved through **Tonal Layering** rather than structural lines.

- **The Layering Principle:** Place a `surface_container_lowest` (#ffffff) card on top of a `surface_container_low` (#f0f4f8) background. This provides a clean, "paper-on-desk" separation.
- **Ambient Shadows:** For floating cards, use an extra-diffused shadow: `box-shadow: 0 12px 40px rgba(11, 33, 71, 0.06);`. The shadow color is a tinted version of our `primary_container` to mimic natural ambient light.
- **The "Ghost Border" Fallback:** If accessibility requires a container edge, use a "Ghost Border": `outline_variant` at 15% opacity. Never use 100% opaque borders.
- **Roundedness:** Use the `lg` (0.5rem / 8px) token for cards and major containers to maintain a professional, slightly softened technical look. Use `full` (9999px) for status chips and action buttons.

---

## 5. Components

### Navigation Sidebar
- **Background:** `primary_container` (#0b2147).
- **Active State:** Use a "Glass" pill background with `primary_fixed_dim` text. Avoid high-contrast boxes; the active item should feel like it's "glowing" within the dark sidebar.

### Data Cards & Sections
- **Construction:** No dividers. Use `spacing-6` (2rem) of vertical white space to separate groups. 
- **Header:** A Title-md header should be left-aligned with an accent icon using a `tertiary_container` (#002819) background for high-end visual interest.

### Interactive Elements
- **Primary Buttons:** Utilize the "Signature Texture" gradient. Padding: `spacing-2.5` (top/bottom) and `spacing-5` (left/right).
- **Status Chips:** High-contrast backgrounds (Emerald/Amber/Blue) at 10% opacity, with the foreground text at 100% opacity for a "soft-glow" technical look.
- **Input Fields:** Use `surface_container_highest` for the input track. Upon focus, transition the "Ghost Border" from 15% to 40% opacity—do not change the background color.

---

## 6. Do's and Don'ts

### Do
- **Do** use whitespace as a functional tool. If data feels crowded, increase the `spacing` scale rather than adding a border.
- **Do** use tonal shifts to indicate "Active" rows in tables (e.g., shifting from `surface` to `surface_container_high`).
- **Do** align all technical labels (Label-sm) to a strict vertical grid to emphasize the "Registry" feel.

### Don't
- **Don't** use pure black (#000000) for text or shadows. Always use the `on_surface` (#171c1f) or a tinted navy.
- **Don't** use sharp 0px corners. Every element must adhere to the **Roundedness Scale** to maintain the premium, modern aesthetic.
- **Don't** use traditional "Drop Shadows" (dark, offset, tight). Only use **Ambient Shadows** (centered, wide-spread, low-opacity).
- **Don't** use horizontal rules (HR) to separate list items. Use the space between items or a subtle background color change on hover.
